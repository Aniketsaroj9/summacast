<objective>
Setup local whisper.cpp executable, download a lightweight transcription model, and create a Node.js background worker skeleton that reads jobs from Redis and runs the transcription pipeline.

Purpose: Offloads CPU-intensive transcription work from the main FastAPI server to a dedicated background process.
Output: setup_whisper.ps1, worker/package.json, worker/transcribe.js, worker/index.js
</objective>

<threat_model>
## Threat Model Analysis (ASVS Level 1)

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Command Injection via file paths in ffmpeg/whisper | High | Pass file path arguments to `spawn` as items in an arguments array. Never use `shell: true` or string concatenation for executing shell commands. Validate that all file paths resolved are within the project workspace bounds. |
| Path Traversal in setup download script | Medium | Restrict download location to `worker/bin` and `worker/models` directories within the project workspace. Prevent download of arbitrary files. |
| Denial of Service (DoS) due to CPU/Memory exhaustion | High | Process transcription tasks sequentially (concurrency limit = 1). If another job arrives, it remains in the Redis queue list until the active job finishes. |
</threat_model>

<task type="auto">
  <name>Task 3: Implement transcribe module</name>
  <files>worker/transcribe.js</files>
  <read_first>backend/storage.py</read_first>
  <action>
    Implement `worker/transcribe.js` which exports two functions:
    1. `extractAudio(inputPath, outputPath)`:
       - Spawns `ffmpeg` (or uses `fluent-ffmpeg`) to extract audio and resample to 16kHz mono WAV (`-vn -acodec pcm_s16le -ac 1 -ar 16000`).
    2. `transcribeAudio(audioPath, modelPath, outputPathWithoutExtension)`:
       - Spawns the whisper.cpp binary (`main.exe` or `whisper-cli.exe` depending on what was downloaded) with arguments `['-m', modelPath, '-f', audioPath, '-oj', '-of', outputPathWithoutExtension]`.
       - Reads the resulting `.json` file, parses the transcript segments, and deletes the temporary WAV file and JSON file.
       - Returns a structured array of segments: `{ start: float, end: float, text: string }`.
  </action>
  <verify>node -e "require('./worker/transcribe')"</verify>
  <acceptance_criteria>
    - `worker/transcribe.js` exists.
    - It exports `extractAudio` and `transcribeAudio` functions.
  </acceptance_criteria>
  <done>Audio extraction and whisper spawning helper functions are implemented.</done>
</task>
