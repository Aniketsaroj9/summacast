<objective>
Integrate FastAPI backend with Redis queue and PostgreSQL database, enabling files uploaded via FastAPI to trigger background transcription, and expose a FastAPI GET endpoint to retrieve the transcript segments.

Purpose: Connects the upload workflow with the transcription worker and exposes the results to the client.
Output: Modifications to backend/main.py, backend/models.py, backend/requirements.txt; new files worker/db.js, updates to worker/index.js
</objective>

<threat_model>
## Threat Model Analysis (ASVS Level 1)

| Threat | Severity | Mitigation |
|--------|----------|------------|
| SQL Injection in database updates | High | Use parameterized queries (using `$1`, `$2` placeholders) in Node.js `pg` client. In FastAPI, utilize SQLAlchemy's ORM model mapping and query parameters to ensure query compilation is safe. |
| Queue Poisoning (Malicious inputs in Redis) | Medium | In the worker, parse the message payload as an integer, validate it exists in the database, and ignore/discard any invalid message that does not correspond to a valid Media record. |
| Insecure Direct Object Reference (IDOR) / Data Exposure | Medium | Ensure that requesting a transcript for a non-existent media file returns a proper 404 Not Found error instead of empty segments or traceback errors. |
</threat_model>
